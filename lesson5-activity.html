Lesson 5: Conduct Testing and Documentation

Activities

Practice Exercise

Preparation:

  • ● System Setup: A pre-assembled computer system or a simulated environment with an intentional issue (e.g., unseated RAM, disconnected storage device).
  • ● Documentation Template: A structured document for students to record:
    • 1. Testing Steps
    • 2. Identified Issue
    • 3. Resolution
    • 4. Recommendations

Procedure:

  • 1. Demonstration (10 minutes)
    • ▪ Power on the system and access the BIOS to check hardware recognition.
    • ▪ Identify an issue in the system (e.g., missing boot device).
    • ▪ Troubleshoot step-by-step (e.g., reconnect the storage device).
    • ▪ Discuss why the issue occurred and how to resolve it.
  • 2. Documentation Task (5 minutes)
    • ● Students observe the demonstration and record their findings.
    • ● Ensure they include testing steps, identified issues, resolutions, and recommendations

Rubrics

Post-Assessment

Directions: Read each question carefully. Choose the best answer from the given options.

1. What is the primary purpose of system testing?
2. What is the most common cause of overheating shutdowns?
3. Which of the following steps is essential in troubleshooting a "No Boot Device Found" error?
4. Which component is responsible for displaying output on the monitor?
5. What is the best way to prevent software errors in a system?
6. What is the function of documentation in troubleshooting?
7. What should you check first when a USB device is not recognized?
8. What tool is commonly used to monitor CPU temperature?
9. Which step should be done first when troubleshooting a system issue?
10. What is the purpose of stress testing in system evaluation?

What I Have learned

Reflective Journal

1. What are the most common errors observed during testing, and what caused them?.

2. How does maintaining accuracy help in identifying and fixing issues?

3. Why is organized documentation important in system testing and troubleshooting?

Wrap-up Discussion:

  • ● Reiterate the importance of testing and documentation in ensuring a system's functionality and long-term usability.
  • ● Encourage students to apply these practices in their future projects.
  • ● Ask: "How does proper documentation help future developers or users?"
